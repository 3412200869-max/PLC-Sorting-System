## 1. 文档目的

本文件定义 PLC 智能分选系统的物理输入、物理输出、仿真信号、HMI 命令、变量命名规则和 TIA Portal 变量组织方式，作为后续状态机、报警清单、PLC 程序结构、HMI 变量映射和仿真测试的输入。

本文件只进行 I/O 与变量设计，不包含 PLC 程序代码。

## 2. 设计范围与假设

### 2.1 系统范围

- 一条输送线。
- 一个入料检测位置。
- 一个分类检测工位。
- A/B 两类物料。
- 一个分选位置和一个弹簧复位式分选执行器。
- 一个出料确认位置。
- 启动、停止、复位、模式、急停状态、报警和状态显示。
- 无实体设备，全部现场信号由 S7-PLCSIM 与 PLC 仿真模型模拟。

### 2.2 地址使用说明

- 本表中的 `I0.0～I1.5`、`Q0.0～Q0.5` 为初版逻辑地址规划。
- CPU 具体订货号、固件和 TIA Portal 设备组态尚未锁定，正式建项时必须根据硬件目录核对起始地址与通道数量。
- 核心控制逻辑不直接依赖绝对地址；物理 I/O 先映射到标准化接口，再由设备和工艺模块使用。
- 仿真信号、HMI 命令、报警与统计不是物理 I/O，不占用 I/Q 地址。
- 所有布尔量按“正逻辑语义”命名，例如 `EStopHealthy=TRUE` 表示急停链正常。

### 3.1 输入设计说明

- `I_ClassASensor` 只在 `I_DetectPosSensor=TRUE` 的有效检测窗口内采样并锁存。
- 当检测位有物料时，分类信号为 TRUE 判定为 A 类，为 FALSE 判定为 B 类；无物料时不得解释为 B 类。
- 分类无效、传感器卡住和信号矛盾通过定时监控及仿真故障变量判定，不额外占用物理 DI。
- 急停状态在项目中仅用于功能逻辑仿真；真实设备必须使用符合风险评估要求的安全回路。

### 5.1 输出设计说明

- 所有最终输出均由对应设备控制模块唯一写入；自动逻辑和手动逻辑只产生请求。
- 急停链异常、关键故障或 PLC 进入停止状态时，`Q_ConveyorRun` 和 `Q_DiverterExtend` 必须回到 0。
- 初版执行器采用单电磁阀、弹簧复位模型，因此只需要一个 DO。若后续改为双电控阀，必须增加缩回输出并设计双线圈互锁。
- 指示灯与蜂鸣器用于工程展示；HMI 状态显示不占用物理输出地址。

### 9.1 信号选择原则

标准化输入由 I/O 映射层选择数据源：

- 仿真关闭：标准输入来自 `I_...` 物理地址。
- 仿真开启：标准输入来自 `Sim_...` 虚拟设备模型。
- 控制核心只使用标准输入，不同时读取物理地址和仿真变量。
- `Sim_Enable` 的切换只允许在系统停止状态下进行。

## 10. PLC 变量命名规范

### 10.1 通用规则

- 变量使用英文，采用 `PascalCase` 主体和明确层级前缀。
- 名称表达真实语义，避免 `M1`、`Sensor1`、`Flag2` 等含义不清的名称。
- 布尔变量优先使用肯定语义，如 `EStopHealthy`、`StartPermitted`。
- 同一对象名称保持一致：输送带统一使用 `Conveyor`，分选执行器统一使用 `Diverter`。
- 不在变量名称中写绝对地址；地址由 TIA Portal 标签表管理。
- 变量名称不使用中文、空格和特殊符号；中文说明填写在 Comment 字段。
- 时间参数写明用途，工程单位填写在注释中，例如 `Cfg_DiverterExtendTime`，单位为 ms。

### 10.2 前缀规范

| 前缀 | 用途 | 示例 |
|---|---|---|
| `I_` | 物理数字量输入 | `I_InfeedSensor` |
| `Q_` | 物理数字量输出 | `Q_ConveyorRun` |
| `AI_` | 物理模拟量输入 | `AI_InspectionValue` |
| `AO_` | 物理模拟量输出 | `AO_ConveyorSpeedRef` |
| `IO_` | 标准化 I/O 接口成员或映射结果 | `IO_In.DetectPosSensor` |
| `HMI_Cmd_` | HMI 操作请求 | `HMI_Cmd_Start` |
| `HMI_Sts_` | HMI 状态发布 | `HMI_Sts_SystemState` |
| `Sim_` | 仿真控制、虚拟输入和故障注入 | `Sim_BlockOutletSensor` |
| `Cfg_` | 可配置工艺参数 | `Cfg_DetectionTimeout` |
| `Alm_` | 报警条件或报警状态 | `Alm_ConveyorFault` |
| `Sts_` | 内部设备或工艺状态 | `Sts_DiverterReady` |
| `Cmd_` | 模块间动作请求 | `Cmd_ConveyorAutoRun` |
| `Cnt_` | 计数值 | `Cnt_ClassA` |

### 10.3 数据类型规范

| 信息类型 | 建议数据类型 | 说明 |
|---|---|---|
| 开关、状态、命令 | Bool | 单一真/假含义 |
| 模式、类别、系统状态 | USInt 或枚举类型 | 使用命名常量/枚举，禁止无说明的魔法数字 |
| 步骤号、报警码 | UInt | 预留明确编号范围 |
| 生产计数 | DInt | 提供足够累计范围 |
| 延时与动作时间 | Time | 注释中写工程单位和合法范围 |
| 速度、比例和统计值 | Real | 明确缩放和单位 |
| 文本显示索引 | UInt/枚举 | PLC 发布代码，HMI 负责多语言文本 |

### 10.4 注释规范

每个 I/O 和接口变量在 TIA Portal 中至少填写：中文功能说明、有效状态、来源/去向和必要的故障安全状态。模拟量还必须填写量程、工程单位、原始范围和缩放方式。

## 11. TIA Portal 变量表结构

### 11.1 PLC 标签表

| 标签表名称 | 内容 | 地址类型 | 维护原则 |
|---|---|---|---|
| `TT_DI_Physical` | DI-001～DI-014 | `%I` | 仅保存物理输入地址、符号和注释 |
| `TT_DO_Physical` | DO-001～DO-006 | `%Q` | 仅保存物理输出地址、符号和注释 |
| `TT_AI_Physical` | 后续 AI 通道 | `%IW` | V1.0 为空，启用时增加缩放说明 |
| `TT_AO_Physical` | 后续 AO 通道 | `%QW` | V1.0 为空，启用时增加缩放说明 |
| `TT_SystemConstants` | 状态码、类别码、报警等级等常量 | 常量 | 使用有意义名称，集中管理 |

### 11.2 数据块规划

| 数据块建议名称 | 内容 | HMI 访问 | 仿真访问 |
|---|---|---|---|
| `DB_IO_Interface` | 标准化输入、最终输出、设备反馈和映射状态 | 只读监视 | 仿真层写标准输入源 |
| `DB_HMI_Interface` | HMI 命令、状态、联锁原因和统计显示 | 读/受控写 | 可用于联合仿真 |
| `DB_Simulation` | 仿真使能、物料生成、虚拟反馈和故障注入 | 调试画面访问 | 读写 |
| `DB_Configuration` | 动作时间、超时、批次目标、上下限 | 按权限读写 | 读取 |
| `DB_Alarm` | 报警条件、活动状态、确认和历史接口 | 读/确认请求 | 故障注入间接触发 |
| `DB_ProductionData` | 物料编号、分类、计数、周期和运行时间 | 只读/受控清零 | 读取 |
| `DB_DataExchange` | 后续 Python 使用的稳定数据快照 | 默认只读 | 不参与设备仿真 |

### 11.3 表与块的边界

- PLC 标签表负责绝对 I/Q 地址绑定。
- 数据块负责程序内部结构化数据、HMI、仿真和外部数据接口。
- HMI 不直接写 `TT_DO_Physical` 中的输出标签。
- 仿真变量不得与物理输入使用相同绝对地址。
- 设备控制、状态机和报警逻辑使用符号变量及结构化接口，不使用散落的 M 区临时位作为长期状态。
