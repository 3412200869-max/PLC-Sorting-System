## 1. 暂停说明

PLC-Sorting-System 已完成阶段2系统详细设计，并进入阶段3开发环境准备。由于当前处于暑假阶段，学校实验室暂不可访问，无法确认课程实际使用的 TIA Portal、S7-PLCSIM 和 WinCC 版本及许可证条件。

为避免安装与学校环境不兼容的软件版本、重复下载大型安装介质或建立无法在实验室复现的TIA工程，项目在正式安装和PLC实现前暂停。

## 3. 暂停原因

当前处于暑假阶段，学校实验室资源暂不可获取，具体影响包括：

- 无法确认学校实际使用的TIA Portal主版本和Update级别。
- 无法确认课程使用的S7-1200 CPU型号及固件。
- 无法确认S7-PLCSIM和WinCC组件、Runtime及许可证可用情况。
- 无法验证个人安装环境与学校工程文件的兼容性。

该原因属于外部资源和时间窗口限制，不是设计缺陷。阶段2文档继续作为正式技术基线保存。

## 4. 暂停期间控制措施

- 不安装未经学校环境确认的TIA Portal主版本。
- 不创建正式TIA工程，不编写PLC控制逻辑。
- 不修改已评审通过的系统架构、I/O、状态机、报警和HMI接口。
- 保持本地Git仓库和项目管理文档完整，不删除阶段成果。
- 如需学习，仅使用不改变项目基线的通用资料；不将试验性代码合并到正式项目。

## 6. 开学后恢复流程

1. 进入学校实验室，记录TIA Portal、STEP 7、S7-PLCSIM和WinCC的完整版本信息。
2. 记录实验室S7-1200 CPU型号、订货号、固件和I/O地址范围。
3. 对照Siemens Compatibility Tool核对个人电脑Windows与学校软件组合。
4. 决定个人环境采用学校版本还是经过批准的兼容版本。
5. 按《PLC开发环境安装执行计划V1.0》完成软件安装和许可证检查。
6. 使用临时空项目验证CPU组态、编译、PLCSIM下载、RUN/STOP和WinCC启动。
7. 更新环境基线、项目状态、README和开发路线图。
8. 关闭阶段3入口门禁后，创建正式TIA Portal工程骨架。
9. 按模块进入PLC程序实现与PLCSIM仿真阶段。

## 7. 恢复后的第一项任务

**确认学校实验室TIA Portal版本及配套环境。**

需要记录：

- TIA Portal主版本和Update。
- STEP 7 Basic/Professional版本与许可证。
- S7-PLCSIM版本。
- WinCC Advanced或Unified产品线、版本和Runtime。
- S7-1200 CPU型号、订货号和固件。

在上述信息确认前，不开始正式PLC程序实现。
