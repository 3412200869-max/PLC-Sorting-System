## 1. 文档目的

本文件定义 PLC 智能分选系统的 WinCC HMI 画面结构、导航、操作权限、显示规则、PLC 通信变量及数据块接口，作为后续 TIA Portal HMI 工程组态、联合仿真和 FAT 测试的输入。

本文件只进行 HMI 与通信接口设计，不生成 PLC 或 HMI 脚本代码。

## 2. 设计依据与边界

设计依据：

- 《PLC 智能分选系统需求分析》PSS-URS-001。
- 《系统总体架构说明》PSS-SAD-001。
- 《PLC 智能分选系统 I/O 点表 V1.0》PSS-IOL-001。
- 《PLC 智能分选系统 自动运行状态机与报警清单 V1.0》PSS-SMAL-001。

设计边界：

- HMI 用于操作请求、状态监视、报警处理、参数设置和生产统计。
- HMI 不直接写入 PLC 的 Q 输出，也不实现工艺顺序、设备互锁或安全逻辑。
- HMI 上的急停图形只显示状态；仿真急停操作只允许出现在明确标识的调试区域，不能替代真实急停装置。
- 历史报警、数据归档、用户管理和画面模板的具体能力取决于实际 WinCC 版本及许可证，正式实施前确认。

## 3. HMI 总体结构

### 3.1 画面层级

```text
P00_GlobalTemplate  全局模板/公共区域
├─ P01_Main         主界面
├─ P02_Manual       手动控制界面
├─ P03_Auto         自动运行界面
├─ P04_Alarm        报警界面
├─ P05_Parameter    参数设置界面
└─ P06_Diagnostics  I/O与通信诊断界面（推荐辅助画面）
```

### 3.2 全局画面模板

所有业务画面复用统一的顶部状态栏、中央内容区和底部导航栏。

| 区域 | 固定内容 | 设计要求 |
|---|---|---|
| 顶部状态栏 | 项目名称、日期时间、登录用户、PLC/HMI通信、当前模式、当前状态 | 所有画面始终可见 |
| 报警横幅 | 最高优先级活动报警、首发报警编号、活动报警数量 | A级报警红色闪烁，确认后常亮；无报警时隐藏 |
| 中央内容区 | 当前业务画面 | 画面对象采用一致的设备符号与文本 |
| 底部导航栏 | 主界面、手动、自动、报警、参数、诊断 | 当前画面按钮显示选中状态 |
| 公共操作区 | 停止、报警确认入口、登录/注销 | 停止入口不应被普通弹窗遮挡 |

### 3.3 导航与弹窗

- 画面切换不改变 PLC 运行状态。
- 参数确认、计数清零、故障复位等高影响操作使用确认弹窗。
- 操作被 PLC 拒绝时，显示返回的联锁原因码和对应说明。
- A 级报警发生时自动显示报警横幅，但不强制切换画面，以免打断正在进行的诊断操作。

## 4. 画面视觉与交互规范

### 4.1 状态颜色

| 状态 | 建议颜色 | 用途 |
|---|---|---|
| 停止/未激活 | 灰色 | 设备未运行、功能未选择 |
| 就绪/正常 | 绿色 | 许可成立、反馈正常 |
| 自动运行 | 绿色或蓝绿色 | 设备正在正常执行 |
| 手动/维护 | 黄色 | 提醒当前处于人工控制模式 |
| 警告 | 黄色/琥珀色 | C级提示或需要关注的状态 |
| 故障/急停 | 红色 | 仅用于实际故障、急停和A/B级报警 |
| 通信未知 | 深灰色并带问号 | 数据无效或通信断开 |

颜色不得作为唯一信息来源，必须同时显示文字、图标或状态码。红色不用于普通启动按钮背景，避免与故障语义冲突。

### 4.2 按钮规则

- 启动、复位、模式选择等按钮向 PLC 写入瞬时请求，PLC 返回接受结果。
- 点动按钮采用“按下请求、释放撤销”，通信丢失时请求应回到 FALSE。
- 不满足权限或 PLC 联锁时按钮禁用，并提供禁用原因。
- 停止按钮保持易见、易操作；停止请求仍由 PLC 按正常停止策略执行。
- 所有数值输入显示工程单位、范围和当前值。

### 4.3 数据刷新建议

| 数据类别 | 建议采集周期 | 示例 |
|---|---:|---|
| 操作和设备状态 | 250～500 ms | 按钮反馈、运行、到位、急停 |
| 状态机与报警摘要 | 500 ms | 当前状态、步骤、首发报警 |
| 参数和计数 | 1 s | 延时、产量、分类数量 |
| 趋势与非关键统计 | 2～5 s | 平均节拍、运行时间 |

最终周期根据 WinCC Runtime、计算机性能和通信负载调整。

## 5. 主界面 `P01_Main`

### 5.1 画面目标

用一屏展示系统身份、总状态、当前模式、设备运行、物料流程、主要报警和生产统计，作为默认启动画面。

### 5.2 画面对象

| 区域 | 显示内容 | 交互 |
|---|---|---|
| 标题区 | `PLC智能分选系统`、项目版本 | 无 |
| 状态卡片 | 当前系统状态、自动/手动模式、启动许可、当前步骤 | 点击联锁原因可进入诊断画面 |
| 工艺总览 | 输送线、入料位、检测位、分选位、A/B料道、执行器 | 点击设备打开状态弹窗，不直接控制 |
| 设备指示 | 输送命令/反馈、执行器原位/伸出、传感器状态 | 状态查看 |
| 物料卡片 | 当前物料编号、类别、当前位置、周期时间 | 状态查看 |
| 生产摘要 | 总数、A类数、B类数、异常数 | 进入统计详细区 |
| 公共操作 | 停止、报警入口 | 停止需要确认当前为正常停止请求 |

### 5.3 动画原则

- 输送线动画由 PLC 返回的运行反馈驱动，而不是由 HMI 启动按钮状态驱动。
- 物料位置由 PLC 当前步骤/位置状态驱动，不在 HMI 内独立计算。
- 分选执行器颜色和位置由原位/伸出反馈决定；反馈矛盾时显示红色故障图标。

## 6. 手动控制界面 `P02_Manual`

### 6.1 进入条件

- 用户至少具有维护/调试权限。
- PLC 处于手动模式或允许切换到手动的停止状态。
- 自动循环未运行。

### 6.2 控制对象

| 功能区 | 显示 | 操作 | PLC 许可条件 |
|---|---|---|---|
| 电机启停 | 输送命令、运行反馈、故障 | 手动启动请求、停止请求 | 手动模式、急停正常、无电机故障 |
| 输送点动 | 入料/检测/分选/出料传感器 | 按住点动 | 手动模式、安全许可成立 |
| 分选执行器 | 原位、伸出到位、动作命令 | 按住伸出、释放返回 | 手动模式、输送停止、反馈不矛盾 |
| 单独测试 | 指示灯、蜂鸣器、虚拟传感器 | 灯测试、蜂鸣器测试、单步仿真 | 调试权限、系统停止或仿真模式 |

### 6.3 安全与互锁显示

- 每个按钮旁显示“允许/禁止”状态。
- 禁止时显示具体原因，例如“自动模式运行中”“急停未释放”“执行器未在原位”。
- HMI 只写手动请求；最终输出由 PLC 设备功能块统一仲裁。
- 手动页面关闭或用户注销时，所有点动请求必须撤销。

## 7. 自动运行界面 `P03_Auto`

### 7.1 主要功能

| 区域 | 内容 | 操作 |
|---|---|---|
| 自动控制 | 自动模式状态、启动许可、运行状态 | 自动模式请求、自动启动、自动停止 |
| 步骤显示 | 当前步骤编号、步骤名称、进入时间、步骤超时剩余/状态 | 无直接步骤跳转 |
| 当前物料 | 物料编号、A/B类别、检测状态、当前位置 | 无直接修改 |
| 流程指示 | 入料、检测、分选、出料各步骤 | 状态显示 |
| 批次统计 | 目标数、完成数、A/B数量、异常数 | 按权限设置目标或请求清零 |
| 运行提示 | 启动拒绝原因、停止待处理、当前报警摘要 | 进入报警/诊断画面 |

### 7.2 自动操作规则

- 自动启动按钮仅在 `StartPermitted=TRUE` 时启用。
- 自动停止按钮产生正常停止请求，不直接关闭输出。
- 不提供 HMI 强制跳步、强制完成或直接修改当前类别的生产操作。
- 调试仿真需要改变类别时，应在独立诊断/仿真区域完成，并清晰标识“仿真”。

## 8. 报警界面 `P04_Alarm`

### 8.1 当前报警

使用 WinCC 报警视图显示：发生时间、报警编号、等级、报警名称、当前状态、确认状态和恢复时间。默认按等级和发生时间排序，A 级优先。

### 8.2 历史报警

在 Runtime 和许可证支持时启用历史报警归档，至少提供：

- 时间范围筛选。
- 报警等级与编号筛选。
- 发生、确认和恢复时间。
- 导出或截图用于 FAT 测试记录。

若当前 WinCC 环境不支持历史归档，V1.0 仍保留画面与接口，并使用活动报警及测试记录替代。

### 8.3 报警操作

| 操作 | 变量/对象 | 处理原则 |
|---|---|---|
| 确认选中报警 | WinCC 报警确认或 PLC 确认请求 | 只改变确认状态，不清除触发条件 |
| 全部确认 | `HMI_Cmd_AlarmAckAll` | 需操作员权限，记录操作结果 |
| 故障复位 | `HMI_Cmd_FaultReset` | PLC 检查条件消失后执行 |
| 消音 | `HMI_Cmd_BuzzerMute` | 只关闭蜂鸣器，不清除报警 |
| 查看处理提示 | 报警编号关联文本 | 显示原因、检查项和恢复条件 |

### 8.4 报警显示规则

- A级：红色，需要停机和人工复位。
- B级：黄色/红色，执行受控停止或阻止新物料。
- C级：黄色提示，不影响基本控制。
- 急停与故障必须区分显示；急停释放后仍需复位恢复。

## 9. 参数设置界面 `P05_Parameter`

### 9.1 参数分组

| 分组 | 参数 | 单位/类型 | 建议范围 | 修改条件 |
|---|---|---|---|---|
| 输送延时 | `Cfg_ConveyorStartTimeout` | Time / s | 0.5～10 s | 停止或待机，维护权限 |
| 输送延时 | `Cfg_ConveyorStopTimeout` | Time / s | 0.5～10 s | 停止或待机，维护权限 |
| 物料延时 | `Cfg_InfeedToDetectTimeout` | Time / s | 1～30 s | 停止或待机，维护权限 |
| 物料延时 | `Cfg_DetectToSortTimeout` | Time / s | 1～30 s | 停止或待机，维护权限 |
| 分选参数 | `Cfg_DiverterExtendTimeout` | Time / s | 0.1～5 s | 停止或待机，维护权限 |
| 分选参数 | `Cfg_DiverterReturnTimeout` | Time / s | 0.1～5 s | 停止或待机，维护权限 |
| 分选参数 | `Cfg_OutletTimeout` | Time / s | 1～30 s | 停止或待机，维护权限 |
| 计数参数 | `Cfg_BatchTarget` | DInt / 件 | 1～100000 | 停止或待机，操作员以上 |

### 9.2 参数修改流程

1. HMI 在编辑区显示当前值、拟修改值、单位和上下限。
2. 用户输入新值，HMI 完成基本格式检查。
3. 用户点击“应用”，写入参数申请值并置位应用请求。
4. PLC 再次检查范围、模式和权限条件，返回成功或拒绝原因。
5. HMI 只在 PLC 确认后更新“当前生效值”。

参数恢复默认值和生产计数清零必须使用独立确认弹窗，不能与普通参数应用共用一个按钮。

## 10. 诊断界面 `P06_Diagnostics`

推荐增加诊断画面，以体现工程调试能力：

- PLC/HMI 通信与心跳状态。
- 14个 DI、6个 DO 的符号名、地址和值。
- 当前启动许可和联锁原因。
- 当前状态、自动步骤和步骤运行时间。
- 仿真使能、下一物料类别及故障注入入口。

仿真和故障注入区域仅对调试权限开放，并使用醒目的“SIMULATION”标识。

## 11. 用户权限规划

| 角色 | 允许功能 | 禁止功能 |
|---|---|---|
| 访客/展示 | 查看主画面、自动状态、报警和统计 | 所有控制与参数修改 |
| 操作员 | 自动启停、报警确认、查看参数、批次操作 | 手动设备测试、故障注入、高级参数 |
| 维护/调试 | 手动点动、参数修改、复位、仿真与诊断 | 修改用户与系统级配置 |
| 管理员 | 用户管理、全部调试和配置 | 仍不能绕过 PLC 安全联锁 |

如果所用 WinCC Runtime 不支持用户管理，至少通过独立调试模式、密码保护或工程版本区分操作与调试功能。

## 12. PLC 与 HMI 通信架构

### 12.1 接口细化决策

总体架构中的概念性 `DB_HMI_Interface` 在本详细设计中拆分为：

```text
DB_HMI_Control      HMI写、PLC读：命令请求与握手
DB_HMI_Status       PLC写、HMI读：状态、步骤、设备和诊断
DB_Alarm            PLC维护、HMI读/确认：报警与摘要
DB_Configuration    PLC维护、HMI受控写：参数申请和生效值
DB_ProductionData   PLC维护、HMI读：物料与生产统计
```

拆分目的：明确读写责任，降低误写风险，便于 WinCC 标签分组和后期权限控制。如果目标 WinCC 版本更适合单一 DB，也可在一个 DB 中使用同名结构分区，但读写边界保持不变。

### 12.2 通信连接

| 项目 | 设计 |
|---|---|
| PLC | Siemens S7-1200 / S7-PLCSIM |
| HMI | TIA Portal 集成 WinCC Runtime/仿真 |
| 连接名称 | 建议 `HMI_Connection_S71200` |
| 变量寻址 | 优先符号化访问 DB 变量 |
| 通信监视 | HMI 心跳计数 + PLC 超时监视 |
| 故障策略 | HMI通信异常时禁止新物料并按配置受控停止；PLC保持独立控制和报警 |

## 13. `DB_HMI_Control` 变量接口

该 DB 由 HMI 写入请求，PLC 读取、校验并通过独立确认字段返回结果。所有请求默认 FALSE。

| 变量名称 | 类型 | 数据来源 | 功能说明 |
|---|---|---|---|
| `StartReq` | Bool | 主/自动画面 | 自动启动请求 |
| `StopReq` | Bool | 全局/自动画面 | 正常停止请求 |
| `FaultResetReq` | Bool | 报警画面 | 故障复位请求 |
| `AlarmAckAllReq` | Bool | 报警画面 | 全部活动报警确认请求 |
| `BuzzerMuteReq` | Bool | 报警画面 | 蜂鸣器消音请求 |
| `SelectAutoReq` | Bool | 自动画面 | 自动模式选择请求 |
| `SelectManualReq` | Bool | 手动画面 | 手动模式选择请求 |
| `ConveyorStartReq` | Bool | 手动画面 | 手动输送启动请求 |
| `ConveyorStopReq` | Bool | 手动画面 | 手动输送停止请求 |
| `ConveyorJogReq` | Bool | 手动画面 | 输送按住点动请求 |
| `DiverterJogReq` | Bool | 手动画面 | 分选执行器按住伸出请求 |
| `LampTestReq` | Bool | 手动/诊断画面 | 指示灯测试请求 |
| `BuzzerTestReq` | Bool | 手动/诊断画面 | 蜂鸣器测试请求 |
| `ResetCountersReq` | Bool | 自动/参数画面 | 生产计数清零请求 |
| `ApplyParametersReq` | Bool | 参数画面 | 参数应用请求 |
| `RestoreDefaultsReq` | Bool | 参数画面 | 恢复默认参数请求 |
| `HMIHeartbeat` | USInt | HMI系统任务 | 每1秒递增或变化，用于PLC监视通信 |

### 13.1 命令握手原则

- 普通命令使用瞬时请求，并由 PLC 进行上升沿识别。
- 对参数应用、复位和清零等重要命令，PLC 返回 `LastCommandCode`、`LastCommandResult` 和 `LastCommandRejectCode`。
- 点动命令不使用锁存；释放按钮、离开画面、注销或通信中断时回到 FALSE。
- PLC 不因同一请求持续为 TRUE 而重复执行动作。

## 14. `DB_HMI_Status` 变量接口

该 DB 由 PLC 写入、HMI 只读。

| 变量名称 | 类型 | 数据来源 | 功能说明 |
|---|---|---|---|
| `SystemState` | USInt/Enum | 状态机 | 当前九状态编号 |
| `PreviousState` | USInt/Enum | 状态机 | 前一状态编号 |
| `OperationMode` | USInt/Enum | 模式管理 | 停止、手动、自动 |
| `AutoStep` | UInt/Enum | 状态机 | 当前自动步骤编号 |
| `AutoStepElapsed` | Time | 状态机 | 当前步骤已运行时间 |
| `StartPermitted` | Bool | 许可汇总 | 是否允许自动启动 |
| `InterlockCode` | UInt | 许可汇总 | 主要启动拒绝/联锁原因 |
| `SystemReady` | Bool | 模式/许可 | 系统就绪状态 |
| `AutoRunning` | Bool | 状态机 | 自动运行状态 |
| `StopPending` | Bool | 状态机 | 正在执行正常停止 |
| `EStopActive` | Bool | 安全状态 | 急停链异常状态 |
| `FaultActive` | Bool | 报警汇总 | 存在A/B级阻塞故障 |
| `ConveyorCmd` | Bool | 输送模块 | 输送最终命令 |
| `ConveyorRunning` | Bool | I/O接口 | 输送运行反馈 |
| `ConveyorFault` | Bool | I/O/报警 | 输送故障状态 |

## 20. 联合仿真验收清单

- [ ] 所有画面可通过统一导航访问，公共状态栏始终可见。
- [ ] HMI 启动请求在启动许可不满足时被 PLC 拒绝并显示原因。
- [ ] 手动点动仅在手动模式和安全许可成立时有效。
- [ ] 离开手动画面或通信中断时点动请求自动撤销。
- [ ] 自动画面正确显示九个系统状态、当前步骤和物料类别。
- [ ] 18项报警均能触发正确文本、等级、确认和恢复状态。
- [ ] 故障复位不能清除仍存在的触发条件。
- [ ] 参数越界或运行中修改被 PLC 拒绝并显示原因。
- [ ] HMI 心跳中断触发 B401 通信报警和受控停止策略。
- [ ] HMI 不能直接写入 `Q_ConveyorRun` 或 `Q_DiverterExtend`。
