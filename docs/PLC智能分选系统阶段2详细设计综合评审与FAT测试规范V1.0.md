## 1. 文档目的

本文件完成阶段 2 详细设计的综合一致性评审，建立需求—功能—变量—测试的双向追溯关系，并规定 PLC 智能分选系统的软件 FAT（Factory Acceptance Test，工厂验收测试）方法、用例、判定标准和记录要求。

本文件不修改既有系统架构，不生成 PLC 代码。所有 FAT 用例均为后续 TIA Portal、S7-PLCSIM 和 WinCC 联合仿真的执行规范。

## 3. 综合设计评审

### 3.1 评审范围

- 需求是否被系统功能和程序模块覆盖。
- I/O、HMI、仿真和数据变量是否具有明确来源与去向。
- 状态机、正常停止、急停、故障和恢复路径是否闭合。
- 18项报警是否具有触发、响应、确认和复位方案。
- OB、FB、DB职责是否与架构分层一致。
- WinCC是否只提交操作请求，不绕过PLC联锁直接写输出。
- 每项关键需求是否至少对应一个FAT用例。

### 3.2 评审结果

| 评审项 | 结果 | 说明 |
|---|---|---|
| 项目范围一致性 | 通过 | 单输送线、A/B两类、单执行器、单件在制品范围一致 |
| PLC控制边界 | 通过 | PLC为唯一控制决策中心，最终输出唯一写入 |
| I/O覆盖 | 通过/待组态确认 | 功能覆盖完整；绝对地址需按目标CPU设备组态复核 |
| 状态机闭合性 | 通过 | 九状态包含正常、停止、急停、故障和恢复路径 |
| 报警覆盖 | 通过 | 急停、传感器、执行器、电机、通信和超时均已覆盖 |

| 版本与许可 | 实施前待关闭 | TIA Portal、PLCSIM、WinCC版本及许可尚未确认 |

### 3.3 接口细化说明

以下差异属于详细设计细化，不改变总体架构：

- 早期概念性 `DB_HMI_Interface` 已在HMI详细设计中拆分为 `DB_HMI_Control` 与 `DB_HMI_Status`。
- 早期 `HMI_Cmd_*` 命名在HMI DB内部细化为 `StartReq`、`StopReq`、`FaultResetReq` 等成员；详细HMI文档为实施命名基线。
- `DB_Alarm.Active[1..18]` 的数组索引与非连续报警编号必须通过固定映射表关联，不按报警编号直接作为数组下标。
- HMI历史报警、用户管理和归档功能依实际WinCC版本/许可证启用；不影响PLC报警触发和FAT核心判定。

## 5. FAT总体规范

### 5.1 FAT目的

验证在无实体设备条件下，PLC控制设计与HMI接口能够满足已批准需求，并对正常运行、受控停止、急停、报警、故障恢复、参数和仿真能力形成可重复证据。

### 5.2 测试范围

- PLC上电初始化、模式与启动许可。
- A/B物料检测、分类、跟踪、分选和计数。
- 手动控制与自动/手动互锁。
- 正常停止、急停和故障状态。
- 18项报警及确认、复位和恢复逻辑。
- WinCC状态、操作、报警、参数和统计画面。
- PLCSIM设备反馈、物料生成和故障注入。
- 通信异常与外部数据接口不影响基本控制的边界。

不在范围：真实电气接线、真实电机和气缸性能、网络抗扰度、机械安全、功能安全认证、现场SAT。

### 5.3 测试环境记录

| 项目 | 记录值 |
|---|---|
| 计算机/操作系统 | 待填写 |

| 测试执行人/日期 | 待填写 |

### 5.4 测试前置条件

- OI-001、OI-002已关闭。
- PLC与HMI工程编译无错误；警告已评估并记录。
- 虚拟CPU可下载、启动、停止和在线监视。
- WinCC可连接PLCSIM，PLC/HMI心跳正常。
- 仿真模式只允许在系统停止时启用。
- 所有测试参数、默认值和超时值已记录。
- 生产计数、活动报警和当前物料已清理到规定初始状态。

### 5.5 测试状态

| 状态 | 定义 |
|---|---|
| Not Run | 尚未执行 |
| Pass | 实际结果完全满足预期结果 |
| Fail | 实际结果不满足预期，必须建立偏差/缺陷记录 |
| Blocked | 因环境、许可或前置缺陷无法执行 |
| N/A | 经评审确认不适用，并记录理由 |

### 5.6 总体验收标准

- 所有P0安全响应、状态机、停止和故障恢复用例必须Pass。
- 所有FR-001～FR-014至少有一个Pass用例。
- 18项报警均完成触发、显示、响应和恢复验证。
- 不允许存在未关闭的严重或高优先级缺陷。
- 允许的低优先级偏差必须有风险说明、责任人和计划关闭版本。
- 计数、状态、报警编号和HMI显示与PLC在线值一致。

### 5.7 测试证据

每个用例至少保存：用例编号、执行时间、输入/故障场景、关键PLC变量、HMI截图、实际结果、判定和偏差编号。急停和输出响应测试应保存触发前后 `SystemState`、`Q_ConveyorRun`、`Q_DiverterExtend` 与报警变量的证据。

## 7. FAT详细测试流程

### 7.1 初始化、模式与启动许可

## 9. 偏差与缺陷管理

| 等级 | 定义 | FAT影响 |
|---|---|---|
| Critical | 急停、输出安全状态或核心控制失效 | 立即停止FAT，不得验收 |
| High | 关键功能、报警或恢复流程不符合需求 | 必须修复并回归后验收 |
| Medium | 非关键功能错误，有可控替代方式 | 评估后修复，相关用例回归 |
| Low | 文本、布局或轻微诊断问题 | 可计划后续修复，但需记录 |

任何修复均应记录影响范围，并至少回归原失败用例及相关状态、报警和HMI用例。
