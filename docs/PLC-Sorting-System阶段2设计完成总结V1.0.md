## 1. 总结目的

本文件对 PLC-Sorting-System 截至 2026-07-12 完成的需求、选型和阶段2详细设计成果进行正式收口，记录已冻结的技术基线、阶段门禁结论、开放事项和下一阶段实施计划。

本次仅完成设计总结与项目状态更新，不生成 PLC 代码，不创建 TIA Portal 工程。

## 2. 项目目标与实现边界

项目以 Siemens S7-1200 和 TIA Portal 为核心，采用 S7-PLCSIM 及条件允许时的 WinCC Runtime/仿真，模拟单输送线、A/B 两类物料、单分选执行器的工业分选单元。

项目用于个人工业自动化学习、工程能力展示和求职作品集。当前无实体 PLC 和现场设备，因此设计重点是需求、架构、I/O、顺序控制、报警、HMI、仿真和测试的完整工程流程，而不是实际接线或现场调试。

## 3. 截至今日完成的设计工作

### 3.1 PLC平台选型

- 对比 Siemens S7-1200 与 S7-1500 的学习难度、工业价值、软件环境和仿真能力。
- 选择 Siemens S7-1200 作为主平台，资源暂按 CPU 1214C 级别规划。
- 确定 TIA Portal、STEP 7、S7-PLCSIM 和 WinCC 的西门子一体化工程路线。
- S7-1500 保留为后续平台迁移和高级功能扩展方向。

### 3.2 系统总体架构

- 建立操作与展示层、PLC控制层、设备与仿真层、数据与智能扩展层。
- 明确 PLC 是唯一控制决策中心，HMI 只提交请求，Python/AI 默认只读。
- 明确仿真输入与物理 I/O 通过标准接口切换，核心控制不依赖信号来源。
- 规定最终输出单一写入、安全隔离、状态驱动和可测试性原则。

### 3.3 I/O点表 V1.0

- 分配 14 个数字量输入和 6 个数字量输出。
- 覆盖急停状态、启动/停止/复位、模式、入料、检测、分类、分选、执行器反馈、输送反馈、故障和出料确认。
- 初版不使用 AI/AO，仅保留模拟检测值和输送速度给定的功能扩展项。
- 建立物理 I/O、HMI命令、仿真信号和标准化接口的分层方案。
- 完成变量命名规范、TIA Portal标签表和DB组织规划。

### 3.4 自动运行状态机

- 建立 9 个主状态：初始、待机、自动运行、物料检测、分选执行、正常停止、急停、故障和复位恢复。
- 明确急停、故障、复位、正常停止和正常工艺转换的优先级。
- 建立单件在制品的检测、分类、跟踪、分选和出料确认流程。
- 明确正常停止处理当前在制品、急停立即撤销输出、复位后禁止自动重启。

### 3.5 报警系统

- 建立 A、B、C 三级报警响应机制。
- 定义 18 项报警，覆盖急停、传感器、执行机构、电机、HMI/Python通信、过程超时、无效分类和非法状态。
- 每项报警均具有稳定编号、PLC变量、触发条件、输出响应、确认和复位策略。
- 明确报警确认不等于故障消失，复位不能清除仍存在的触发条件。

### 3.6 PLC逻辑结构规划

- 规划 OB1 主循环、OB100 启动初始化和可选周期任务。
- 规划模式、状态机、输送、检测、物料跟踪、分选、报警、仿真、统计和HMI接口功能块。
- 规划 I/O、HMI控制、HMI状态、仿真、配置、报警、状态机、生产数据和外部数据交换DB。
- 明确OB1调用顺序和最终输出唯一映射位置。

### 3.7 WinCC HMI设计

- 设计主界面、手动控制、自动运行、报警、参数设置五个必需画面及一个推荐诊断画面。
- 规定全局状态栏、报警横幅、导航、颜色、按钮、权限和通信刷新策略。
- 建立 `DB_HMI_Control`、`DB_HMI_Status`、`DB_Alarm`、`DB_Configuration` 和 `DB_ProductionData` 接口。
- 建立命令握手、通信心跳、参数申请—校验—生效和用户角色方案。

### 3.8 FAT测试规范

- 完成阶段2详细设计综合一致性评审，结论为有条件通过。
- 建立 14 项功能需求、6 项非功能需求和 18 项报警的测试追溯关系。
- 设计初始化、模式、正常运行、正常停止、急停、手动、报警、故障恢复、参数、权限、通信和仿真测试。
- 规定FAT环境、前置条件、证据、通过标准、偏差等级和回归要求。

## 9. 风险与控制

| 风险 | 控制措施 |
|---|---|
| TIA/PLCSIM/WinCC版本不兼容 | 先记录实际安装版本并验证联合启动，再创建正式工程 |
| CPU地址与设计假设不一致 | 以设备组态为准调整绝对地址，保持符号接口不变 |
| WinCC许可不支持部分功能 | 保留画面与接口设计，记录替代验证方案 |
| 仿真与真实设备存在差异 | 明确FAT为软件验证，不宣称完成真实SAT或功能安全认证 |
| TIA工程二进制不易审查 | 同步归档包、导出源文件、版本说明和Git提交号 |
