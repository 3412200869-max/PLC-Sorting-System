## 1. 文档目的

本文件定义 PLC 智能分选系统的主状态机、状态转换优先级、正常停止、急停、故障与复位恢复流程，建立报警编号和处理策略，并规划 OB、FB、DB 的软件逻辑结构。

本文件只描述控制设计，不包含 PLC 程序代码。

## 2. 设计输入与范围

设计依据：

- 《PLC 智能分选系统需求分析》PSS-URS-001。
- 《系统总体架构说明》PSS-SAD-001。
- 《PLC 智能分选系统 I/O 点表 V1.0》PSS-IOL-001。

初版范围为单输送线、一个检测工位、A/B 两类物料、一个弹簧复位式分选执行器和一个出料确认传感器。一次只跟踪一件在制物料，多物料队列不在 V1.0 范围内。

## 3. 状态机总体原则

### 3.1 状态变量

| 项目 | 设计值 |
|---|---|
| 主状态变量 | `Sts_SystemState` |
| 建议数据类型 | USInt 或枚举类型 |
| 状态保持 | 保存在状态机实例数据中 |
| 上电默认值 | `0=INITIAL` |
| HMI 发布变量 | `HMI_Sts_SystemState` |
| 非法状态处理 | 立即转入 `70=FAULT`，产生内部状态异常报警 |

### 3.2 状态编号

| 编号 | 符号名称 | 中文名称 | 主要目的 |
|---:|---|---|---|
| 0 | `INITIAL` | 初始状态 | 上电初始化并检查基础条件 |
| 10 | `STANDBY` | 待机状态 | 输出安全、等待模式与启动请求 |
| 20 | `AUTO_RUN` | 自动运行状态 | 输送运行并等待新物料进入 |
| 30 | `MATERIAL_DETECT` | 物料检测状态 | 将物料送至检测位并锁存 A/B 类别 |
| 40 | `SORT_EXECUTE` | 分选执行状态 | 跟踪物料到分选/出口并执行分类 |
| 50 | `NORMAL_STOP` | 正常停止状态 | 禁止新物料并受控处理当前在制品 |
| 60 | `EMERGENCY_STOP` | 急停状态 | 立即撤销运动输出并锁定运行 |
| 70 | `FAULT` | 故障状态 | 按故障策略停机并等待处理 |
| 80 | `RESET_RECOVERY` | 复位恢复状态 | 校验故障消失、清理状态并恢复待机 |

### 3.3 状态转换优先级

每次 PLC 扫描周期按以下优先级判断全局转换：

1. **急停优先级最高：** `I_EStopHealthy=FALSE` 时，无论当前状态为何均转入 `EMERGENCY_STOP`。
2. **关键故障次优先：** 存在 A 级停机故障时转入 `FAULT`；急停状态除外。
3. **复位恢复：** 急停已释放或故障条件已消失，并收到复位请求时转入 `RESET_RECOVERY`。
4. **正常停止：** 自动流程收到停止请求时转入 `NORMAL_STOP`。
5. **正常工艺转换：** 仅在无更高优先级条件时执行当前状态的正常转换。

### 3.4 状态机总览

```text
上电
 │
 ▼
[0 初始] ──初始化通过──> [10 待机] ──自动模式+启动许可+启动请求──> [20 自动运行]
                           ▲                                      │
                           │                                      │检测到入料
                           │                                      ▼
                           │                               [30 物料检测]
                           │                                      │分类有效
                           │                                      ▼
                           │                               [40 分选执行]
                           │                                      │出料确认
                           └────────────无停止请求─────────────────┘

自动相关状态 ──停止请求──> [50 正常停止] ──在制品处理完成/无在制品──> [10 待机]

任意状态 ──急停──> [60 急停] ──释放+复位请求──> [80 复位恢复] ──检查通过──> [10 待机]
任意状态 ──关键故障──> [70 故障] ──故障消失+复位请求──> [80 复位恢复]
```

## 4. 状态详细设计

### 4.1 状态 0：初始状态 `INITIAL`

| 项目 | 设计内容 |
|---|---|
| 进入条件 | PLC 上电、程序下载后首次运行、检测到非法状态编号后重新初始化 |
| 执行动作 | 撤销输送和分选输出；清除瞬时命令；将仿真自动运行置为停止；初始化步骤、定时监控和非保持过程变量；保持需要追溯的累计统计 |
| 正常转换 | 初始化完成、急停链正常、无 A 级故障 → `STANDBY` |
| 异常转换 | 急停链异常 → `EMERGENCY_STOP`；初始化检查失败或状态数据异常 → `FAULT` |
| 退出要求 | 所有运动输出为 0，当前物料状态为“无有效在制品” |

### 4.2 状态 10：待机状态 `STANDBY`

| 项目 | 设计内容 |
|---|---|
| 进入条件 | 初始化完成；正常停止完成；复位恢复检查通过 |
| 执行动作 | 输送停止；分选执行器保持安全原位；不接收物料；计算并发布启动许可；允许模式选择和手动调试请求 |
| 正常转换 | 自动模式有效、启动许可成立、收到启动请求 → `AUTO_RUN` |
| 异常转换 | 急停链异常 → `EMERGENCY_STOP`；出现 A 级故障 → `FAULT` |
| 保持条件 | 未收到有效启动请求，或启动条件不满足 |

启动许可至少包括：急停链正常、无 A/B 级阻塞故障、分选执行器在原位、输送无故障、无未处理在制品、仿真模式切换状态合法。

### 4.3 状态 20：自动运行状态 `AUTO_RUN`

| 项目 | 设计内容 |
|---|---|
| 进入条件 | 待机状态下有效启动；完成一件物料且无停止请求 |
| 执行动作 | 置自动运行标志；允许输送带运行；等待入料传感器上升沿；拒绝手动动作；监视输送反馈和急停/故障 |
| 正常转换 | 检测到新的入料上升沿并成功建立物料记录 → `MATERIAL_DETECT` |
| 停止转换 | 收到正常停止请求且无在制品 → `NORMAL_STOP` |
| 异常转换 | 急停 → `EMERGENCY_STOP`；电机故障、运行反馈超时或其他 A 级故障 → `FAULT` |
| 保持条件 | 自动许可持续成立且尚未检测到入料 |

### 4.4 状态 30：物料检测状态 `MATERIAL_DETECT`

| 项目 | 设计内容 |
|---|---|
| 进入条件 | 自动运行状态检测到有效入料并创建当前物料 |
| 执行动作 | 输送物料至检测工位；启动检测到位超时；在检测窗口读取并锁存 `I_ClassASensor`；赋予类别 A 或 B；记录检测结果有效性 |
| 正常转换 | 检测位到达、分类结果有效并完成锁存 → `SORT_EXECUTE` |
| 停止转换 | 收到停止请求时记录“停止待处理”，继续当前物料至安全处理路径；完成分类后转入 `NORMAL_STOP` 的在制品处理阶段 |
| 异常转换 | 检测位到达超时、传感器卡住或分类无效 → `FAULT`；急停 → `EMERGENCY_STOP` |
| 保持条件 | 物料尚未到达检测位且未超时 |

分类规则：检测窗口内 `I_ClassASensor=TRUE` 锁存为 A 类，FALSE 锁存为 B 类。检测窗口以外不得更新当前物料类别。

### 4.5 状态 40：分选执行状态 `SORT_EXECUTE`

| 项目 | 设计内容 |
|---|---|
| 进入条件 | 当前物料分类有效，已离开检测状态 |
| 执行动作 | 输送并跟踪物料到分选位置；A 类到位后停止输送并驱动分选执行器伸出，再确认返回原位；B 类保持执行器原位并沿主线通过；等待出料确认并更新统计 |
| 正常转换 | 出料确认有效、计数完成、当前物料数据清除；无停止待处理 → `AUTO_RUN` |
| 停止转换 | 当前物料完成且存在停止待处理 → `NORMAL_STOP` |
| 异常转换 | 分选位超时、执行器伸出/返回超时、反馈矛盾、出料超时 → `FAULT`；急停 → `EMERGENCY_STOP` |
| 保持条件 | 当前物料尚未完成，且所有动作未超时 |

V1.0 采用一次一件物料策略：当前物料未完成前不建立下一件物料记录。

### 4.6 状态 50：正常停止状态 `NORMAL_STOP`

| 项目 | 设计内容 |
|---|---|
| 进入条件 | 自动状态收到停止请求；停止待处理物料完成；通信异常策略要求受控停止 |
| 执行动作 | 禁止建立新物料；若无在制品，停止输送并撤销分选命令；若存在已识别在制品且设备无故障，按受控子步骤送至出口；清除自动运行标志 |
| 正常转换 | 无在制品、输送已停止、分选执行器原位 → `STANDBY` |
| 异常转换 | 急停 → `EMERGENCY_STOP`；受控停止过程中出现关键故障 → `FAULT` |
| 保持条件 | 等待当前在制品完成或设备到达安全状态 |

正常停止不得自动复位报警，也不得在输出尚未确认安全时直接宣布待机。

### 4.7 状态 60：急停状态 `EMERGENCY_STOP`

| 项目 | 设计内容 |
|---|---|
| 进入条件 | `I_EStopHealthy=FALSE`，或仿真急停链被断开 |
| 执行动作 | 立即撤销 `Q_ConveyorRun` 和 `Q_DiverterExtend`；禁止自动/手动动作；保持急停报警；标记当前物料为状态未知；HMI 显示急停及恢复提示 |
| 恢复转换 | 急停链恢复正常、其他关键故障不存在、收到人工复位请求 → `RESET_RECOVERY` |
| 异常转换 | 急停链仍不健康时保持本状态；释放后若存在其他 A 级故障则转入 `FAULT` |
| 禁止行为 | 急停释放后自动重启；HMI 启动命令直接离开急停状态 |

急停在本项目中为功能响应仿真，不能替代真实设备的安全继电器、故障安全 PLC 或硬接线安全回路。

### 4.8 状态 70：故障状态 `FAULT`

| 项目 | 设计内容 |
|---|---|
| 进入条件 | 存在 A 级停机故障、非法状态、关键反馈超时或无法继续安全运行的过程异常 |
| 执行动作 | 根据故障策略停止输送和分选输出；锁定自动启动；保持需人工确认的报警；保存故障发生时状态、步骤、物料和关键反馈快照 |
| 恢复转换 | 所有阻塞故障条件消失且收到人工复位请求 → `RESET_RECOVERY` |
| 特殊转换 | 故障状态中急停触发 → `EMERGENCY_STOP` |
| 保持条件 | 故障条件仍存在、未确认或未收到复位请求 |

### 4.9 状态 80：复位恢复状态 `RESET_RECOVERY`

| 项目 | 设计内容 |
|---|---|
| 进入条件 | 急停释放或故障条件消失后收到有效复位请求 |
| 执行动作 | 保持运动输出为 0；清除允许复位的锁存报警；清除瞬时命令、定时器和停止待处理标志；将未完成物料标记为废弃/未确认；验证分选执行器原位和输送停止反馈 |
| 正常转换 | 急停链健康、无阻塞故障、执行器原位、输送已停止、恢复检查全部通过 → `STANDBY` |
| 异常转换 | 恢复检查超时或反馈不满足 → `FAULT`；急停再次触发 → `EMERGENCY_STOP` |
| 禁止行为 | 复位后直接进入自动运行；清除仍存在的故障条件 |

## 5. 正常停止与复位策略

### 5.1 正常停止策略

- 停止请求立即禁止接收新物料。
- 无在制品时停止输送并进入待机。
- 已识别且设备状态正常的在制品允许完成到出口，之后停止。
- 停止过程中出现急停或关键故障时，不再坚持完成物料，立即执行更高优先级响应。
- V1.0 一次只处理一件物料，因此无需清空多件队列。

### 5.2 复位恢复流程

1. 操作员排除报警原因或释放急停。
2. PLC 确认触发条件已经消失。
3. 操作员发出复位请求。
4. 系统进入 `RESET_RECOVERY`，运动输出保持关闭。
5. 清除允许复位的报警和过程临时状态。
6. 检查输送停止、执行器原位及传感器合理性。
7. 检查通过后进入 `STANDBY`。
8. 操作员重新选择自动模式并再次启动；禁止自动重启。

## 6. 报警系统设计原则

### 6.1 报警等级

| 等级 | 名称 | 系统响应 | 确认与复位 |
|---|---|---|---|
| A | 立即停机报警 | 立即撤销相关运动输出并进入急停或故障状态 | 条件消失后人工确认并复位 |
| B | 受控停机报警 | 禁止新物料，按条件完成或停止当前过程 | 条件消失后人工确认/复位 |
| C | 提示报警 | 不影响 PLC 基本控制，仅提示诊断或扩展功能异常 | 可确认；条件恢复后自动消失或记录历史 |

### 6.2 报警变量规则

- 报警布尔变量格式：`Alm_A001_EStopActive`。
- 报警编号保持稳定，显示文本和多语言信息由 HMI 根据编号管理。
- 报警管理区保存触发状态、锁存状态、确认状态、发生时间和恢复状态。
- 复位只能清除触发条件已经消失的锁存报警。

## 7. 报警清单 V1.0

| 编号 | 等级 | 报警名称 | 触发条件 | PLC 变量名称 | 处理方式 |
|---|---|---|---|---|---|
| A001 | A | 急停已触发 | `I_EStopHealthy=FALSE` | `Alm_A001_EStopActive` | 立即撤销所有运动输出，进入急停状态；释放急停并人工复位后恢复 |
| A101 | B | 入料传感器卡住 | `I_InfeedSensor` 连续有效超过允许时间，且无合理物料流程 | `Alm_A101_InfeedSensorStuck` | 禁止新物料，受控停止；检查传感器/仿真信号后复位 |
| A102 | A | 检测位传感器异常 | 检测位信号卡住，或与入料/物料状态矛盾 | `Alm_A102_DetectSensorFault` | 停止输送，进入故障状态；排除信号异常后复位 |
| A103 | A | 分选位传感器异常 | 分选位信号卡住，或无有效物料时异常触发 | `Alm_A103_SortSensorFault` | 停止输送和分选，进入故障状态；检查后复位 |
| A104 | A | 出料传感器异常 | 出料信号卡住，或无在制品时重复触发 | `Alm_A104_OutletSensorFault` | 停止当前循环，进入故障状态；检查后复位 |
| A201 | A | 分选执行器反馈矛盾 | `I_DiverterHome=TRUE` 且 `I_DiverterExtended=TRUE` | `Alm_A201_DiverterFeedbackConflict` | 立即撤销分选输出和输送，进入故障状态；修复反馈后复位 |
| A202 | A | 分选执行器伸出超时 | 已发出伸出命令，超过 `Cfg_DiverterExtendTimeout` 未到位 | `Alm_A202_DiverterExtendTimeout` | 撤销输出、停止输送并进入故障状态；检查执行器后复位 |
| A203 | A | 分选执行器返回超时 | 撤销伸出命令后，超过 `Cfg_DiverterReturnTimeout` 未回原位 | `Alm_A203_DiverterReturnTimeout` | 停止输送并进入故障状态；检查原位反馈后复位 |
| A301 | A | 输送电机故障 | `I_ConveyorFault=TRUE` | `Alm_A301_ConveyorFault` | 立即撤销输送和分选输出，进入故障状态；故障消失后人工复位 |
| A302 | A | 输送启动反馈超时 | 发出运行命令后超过 `Cfg_ConveyorStartTimeout`，`I_ConveyorRunning` 仍为 FALSE | `Alm_A302_ConveyorStartTimeout` | 撤销输送命令并进入故障状态；检查驱动/仿真反馈后复位 |
| A303 | A | 输送停止反馈异常 | 撤销运行命令后超过 `Cfg_ConveyorStopTimeout`，运行反馈仍为 TRUE | `Alm_A303_ConveyorStopFault` | 锁定启动并进入故障状态；检查驱动反馈后复位 |
| B401 | B | HMI 通信异常 | HMI 心跳在 `Cfg_HMICommTimeout` 内未变化，且系统配置要求监视 HMI | `Alm_B401_HMICommFault` | 禁止新物料并执行受控停止；PLC 保持独立诊断，通信恢复后确认/复位 |
| C402 | C | Python 数据接口通信异常 | 启用数据接口后，Python/网关心跳超时 | `Alm_C402_DataInterfaceFault` | 只提示并记录，不影响 PLC 基本控制；通信恢复后自动恢复 |
| A501 | A | 物料到达检测位超时 | 建立物料后超过 `Cfg_InfeedToDetectTimeout` 未检测到检测位信号 | `Alm_A501_DetectArrivalTimeout` | 停止输送并进入故障状态；检查物料或传感器后复位 |
| A502 | A | 物料到达分选位超时 | 分类完成后超过 `Cfg_DetectToSortTimeout` 未检测到分选位信号 | `Alm_A502_SortArrivalTimeout` | 停止输送并进入故障状态；检查物料跟踪后复位 |
| A503 | A | 出料确认超时 | 分选动作/通过动作完成后超过 `Cfg_OutletTimeout` 未收到出料确认 | `Alm_A503_OutletTimeout` | 停止输送并进入故障状态；检查堵料或传感器后复位 |
| A504 | A | 分类结果无效 | 检测窗口结束时未形成有效 A/B 类别，或仿真注入无效分类 | `Alm_A504_ClassificationInvalid` | 停止当前物料处理并进入故障状态；检查检测逻辑后复位 |
| A901 | A | 状态机状态非法 | `Sts_SystemState` 不属于已定义编号 | `Alm_A901_InvalidState` | 立即撤销运动输出并进入故障状态；记录诊断快照后复位 |

## 8. 报警响应矩阵

| 报警类别 | 输送输出 | 分选输出 | 新物料 | 主状态 | 恢复方式 |
|---|---|---|---|---|---|
| 急停 | 立即 OFF | 立即 OFF | 禁止 | `EMERGENCY_STOP` | 释放 + 人工复位 |
| 传感器/执行器/电机 A 级故障 | OFF | OFF | 禁止 | `FAULT` | 条件消失 + 确认/复位 |
| 过程超时 | OFF | OFF | 禁止 | `FAULT` | 排障 + 确认/复位 |
| HMI 通信 B 级故障 | 受控停止 | 按受控停止需要 | 禁止 | `NORMAL_STOP` 或 `FAULT` | 通信恢复 + 确认 |
| Python 接口 C 级故障 | 保持 | 保持 | 允许 | 原状态 | 自动恢复/确认历史 |

## 9. 报警参数规划

| 参数名称 | 用途 | 初版建议 | 最终确定方式 |
|---|---|---:|---|
| `Cfg_ConveyorStartTimeout` | 输送启动反馈等待 | 2 s | PLCSIM 测试后调整 |
| `Cfg_ConveyorStopTimeout` | 输送停止反馈等待 | 2 s | PLCSIM 测试后调整 |
| `Cfg_InfeedToDetectTimeout` | 入料到检测位最大时间 | 5 s | 虚拟输送节拍确定 |
| `Cfg_DetectToSortTimeout` | 检测位到分选位最大时间 | 5 s | 虚拟输送节拍确定 |
| `Cfg_DiverterExtendTimeout` | 执行器伸出最大时间 | 1 s | 仿真动作时间确定 |
| `Cfg_DiverterReturnTimeout` | 执行器返回最大时间 | 1 s | 仿真动作时间确定 |
| `Cfg_OutletTimeout` | 分选后出料确认最大时间 | 5 s | 虚拟输送节拍确定 |
| `Cfg_HMICommTimeout` | HMI 心跳超时 | 5 s | HMI 刷新周期确定 |

以上数值用于初版仿真，不代表真实机械设备参数。所有时间值应配置合法上下限。

## 10. PLC 程序逻辑结构规划

### 10.1 调用层次

```text
OB100 启动初始化
  └─ 初始化请求与上电状态设置

OB1 循环主程序
  ├─ I/O与仿真信号映射
  ├─ HMI命令处理
  ├─ 设备状态与故障诊断
  ├─ 报警汇总与全局许可
  ├─ 模式与主状态机
  ├─ 工艺设备控制
  ├─ 最终输出映射
  └─ HMI、统计与数据发布

可选周期中断OB
  └─ 仅在确有固定周期采样需求时使用，V1.0不强制
```

### 10.2 OB 组织块规划

| 组织块 | 建议名称 | 职责 | 设计约束 |
|---|---|---|---|
| OB1 | `MainCycle` | 按固定顺序调用映射、设备、报警、状态机、输出和数据模块 | 不堆积具体设备细节，不直接多点写输出 |
| OB100 | `Startup` | 设置上电初始化请求，使主状态进入 `INITIAL` | 不在此直接启动设备 |
| 可选周期 OB | `CyclicDataTask` | 后续固定周期统计或数据采样 | 仅确认必要且 CPU 支持后启用 |

### 10.3 FB 功能块规划

| 功能块 | 建议名称 | 主要职责 | 主要实例数据 |
|---|---|---|---|
| FB01 | `FB_ModeManager` | 模式互斥、启动/停止/复位请求及启动许可 | 当前模式、请求沿、拒绝原因 |
| FB02 | `FB_StateMachine` | 主状态、状态进入动作、转换和停止待处理 | 当前/前一状态、进入脉冲、在制品标志 |
| FB03 | `FB_Conveyor` | 输送请求仲裁、输出、反馈和启停超时 | 运行状态、启动/停止定时状态 |
| FB04 | `FB_Detection` | 检测窗口、分类锁存和有效性判断 | 检测状态、锁存类别、检测超时 |
| FB05 | `FB_MaterialTracking` | 单件物料编号、类别、位置和完成事件 | 当前物料记录、位置和时间戳 |
| FB06 | `FB_Diverter` | 分选请求仲裁、伸出/返回反馈及超时 | 执行器状态、动作阶段和定时状态 |
| FB07 | `FB_AlarmManager` | 报警触发、锁存、确认、复位、等级与汇总 | 报警状态、确认状态、首发报警 |
| FB08 | `FB_Simulation` | 虚拟物料、传感器、反馈和故障注入 | 仿真步骤、虚拟位置和时间 |
| FB09 | `FB_Statistics` | A/B/总数、异常数、周期和运行时间 | 计数器、周期统计 |
| FB10 | `FB_HMIInterface` | HMI 命令握手、状态发布和通信心跳 | 命令沿、心跳监视、接口状态 |

### 10.4 FC/辅助功能规划

| 功能 | 建议名称 | 职责 |
|---|---|---|
| 输入映射 | `FC_MapInputs` | 在物理 I/O 与仿真数据源之间选择，生成标准输入 |
| 输出映射 | `FC_MapOutputs` | 将设备模块最终输出映射到 Q 地址 |
| 启动许可汇总 | `FC_EvaluatePermissives` | 汇总急停、故障、设备状态和模式条件 |
| 报警文本索引 | `FC_AlarmCodeSelect` | 生成首发/最高优先级报警编号供 HMI 显示 |

辅助功能是否使用 FC 或在相关 FB 内实现，应在 TIA Portal 详细设计时统一，避免过度拆分。

### 10.5 DB 数据块规划

| 数据块 | 内容 | 主要访问方 |
|---|---|---|
| `DB_IO_Interface` | 标准输入、最终输出和映射状态 | I/O 映射、所有设备 FB |
| `DB_HMI_Interface` | HMI 命令、状态、步骤、联锁原因和统计 | HMI、模式/状态/报警模块 |
| `DB_Simulation` | 仿真使能、虚拟信号和故障注入 | `FB_Simulation`、调试 HMI |
| `DB_Configuration` | 时间参数、阈值、合法范围和默认值 | 设备 FB、状态机、HMI |
| `DB_Alarm` | 报警条件、锁存、确认、等级和摘要 | `FB_AlarmManager`、HMI |
| `DB_StateMachine` | 状态机实例数据或状态接口 | `FB_StateMachine`、HMI |
| `DB_ProductionData` | 当前物料、产量、类别、周期和运行时间 | 跟踪、统计、HMI |
| `DB_DataExchange` | Python 后续使用的只读数据快照 | 数据发布、外部接口 |
| 各实例 DB | 各 FB 的内部状态和定时数据 | 对应 FB |

### 10.6 OB1 建议调用顺序

1. 输入与仿真映射。
2. HMI 命令和通信心跳处理。
3. 输送、检测、分选设备反馈诊断。
4. 报警条件计算与报警汇总。
5. 急停、故障、启动许可和模式管理。
6. 主状态机与物料流程决策。
7. 输送和分选设备输出仲裁。
8. 最终输出映射。
9. 报警、状态、统计和数据接口发布。

该顺序确保本扫描周期的输入和故障先被处理，再决定状态与输出；最终输出只有一个映射位置。

## 11. 设计验证清单

- [ ] 每个定义状态均有进入、保持、正常退出和异常退出条件。
- [ ] 任一运行状态触发急停均能转入 `EMERGENCY_STOP`。
- [ ] 急停释放或故障消失后不能自动重新启动。

- [ ] 每个最终输出只有一个设备 FB 和一个输出映射位置。
- [ ] 状态、报警、设备和统计数据均可通过 HMI 接口观察。
